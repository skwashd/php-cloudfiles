Page Details:

CloudFiles HTTP Client

This is an HTTP client class for Cloud Files. It uses PHP's cURL module to handle the actual HTTP request/response. This is NOT a generic HTTP client class and is only used to abstract out the HTTP communication for the PHP Cloud Files API.

This module was designed to re-use existing HTTP(S) connections between subsequent operations. For example, performing multiple PUT operations will re-use the same connection.

This modules also provides support for streaming content into and out of Cloud Files. The majority (all?) of the PHP HTTP client modules expect to read the server's response into a string variable. This will not work with large files without killing your server. Methods like, get_object_to_stream() and put_object() take an open filehandle argument for streaming data out of or into Cloud Files.
